smart homeevents

Robonomics IPFS 그리고 Home Assistant를 사용하여 완전히 분산된 미디어 파이프라인을 선보이세요 재미있는 사진을 캡처하고 실시간 컴퓨터 비전 필터를 적용하며 IPFS에 저장하고 모든 미디어 해시를 Robonomics Datalog에 등록한 다음 어떤 DApp이든 사진 갤러리를 읽고 표시할 수 있도록 하세요

아이디어

1. 사진 캡처 & 얼굴 감지

카메라가 Home Assistant 환경에 통합되어 필요할 때 이미지를 캡처하고 내장된 Python 서비스가 OpenCV의 Haar-cascade 분류기(GPU 불필요)로 모든 얼굴을 감지하며 감지된 각 얼굴은 Pillow 라이브러리를 통해 웃는 이모지로 덮어씌워 익명화됩니다

2. Home Assistant와 통합

Home Assistant 내에서 배포를 간소화하기 위해 Home Assistant와의 Robonomics 통합의 주요 기여자인 Alena가 주요 Robonomics HA 통합을 기반으로 경량화된 “media integration” 모듈을 만들었습니다 (https://github.com/airalab/homeassistant-robonomics-integration) 패키지 (https://github.com/LoSk-p/robonomics_media_integration) 카메라 이벤트 구독과 메타 파일 CID를 Robonomics Datalog에 게시하기 위해 코어 커넥터를 재사용하지만 사용하지 않는 기능은 제거하여 설치와 설정이 몇 분 만에 끝났습니다

3. IPFS Storage

처리 후 각 이미지는 Pinata의 핀닝 서비스를 사용하여 IPFS에 고정되며 이는 스냅샷의 영구 참조 역할을 하는 고유한 콘텐츠 식별자(CID)를 반환합니다

4. 메타데이터 파일 & 데이터로그

새 사진이 추가될 때마다 시스템은 모든 이미지 CID의 JSON 목록을 다시 만들고 그 목록을 IPFS에 고정합니다 백그라운드 작업이 1분마다 실행되어 최신 고정 목록의 CID를 Robonomics Datalog에 기록된 것과 비교하고 일치하지 않으면 새로운 목록의 CID를 Datalog에 보내 온체인 참조를 최신 상태로 유지합니다

RoSeMAN 색인

RoSeMAN 우리 오프체인 인덱서(센서맵 케이스에서도 사용됨)는 Node.js 서비스입니다(참조 (https://github.com/airalab/RoSeMAN) Substrate 기반 센서 에이전트를 통해 Robonomics Datalog을 구독하는

_

각 새로운 메타 파일 CID가 Datalog에 기록될 때마다 IPFS에서 HTTP fetch를 수행하여 사진 CID의 JSON 목록을 가져오고 파싱합니다

_

파싱된 CID는 RoSeMAN의 MongoDB 백엔드에 삽입되고 간단한 HTTP API를 통해 노출되므로 외부 DApp은 IPFS에 고정된 모든 사진의 최신 갤러리를 RoSeMAN에 쿼리할 수 있습니다
