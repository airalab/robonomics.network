Marco de middleware de Robonomics para ROS 2

robotics

Robonomics es una plataforma de código abierto que facilita tanto servicios simples de IoT como robótica compleja, permitiendo una comunicación efectiva entre agentes, ya sean humanos o robots. Para integrar robots en este sistema, utilizamos ROS. En este artículo, nuestro objetivo es proporcionar una visión general de nuestro proceso para integrar Robonomics en ROS 2.

¿Por qué ROS 2 en Robonomics?

ROS, el Robot Operating System, es uno de los marcos de trabajo de código abierto más antiguos para la robótica. Es una colección de bibliotecas de software y herramientas para hacer que los robots funcionen. Tiene todo, desde lo básico hasta características avanzadas y viene con herramientas útiles para programadores. Es ampliamente utilizado por universidades de todo el mundo para la enseñanza e investigación en robótica, por lo que está bien establecido, funciona con muchos tipos de robots y tiene una gran comunidad de desarrolladores.

Desde que ROS se inició en 2007, han tenido lugar cambios significativos en el campo de la robótica y dentro de la comunidad de ROS. Como respuesta a estos avances, se introdujo ROS 2 con el objetivo de adaptarse a estos cambios. Se basa en las fortalezas de ROS 1 mientras aborda sus limitaciones. Hemos estado utilizando ROS en Robonomics desde el principio, y ahora es el momento de migrar a ROS 2.

Paquetes de Robonomics para ROS y ROS 2

Construir un sistema multi-robótico a menudo implica el desafío de conectar dispositivos de varios proveedores, o incluso dispositivos dentro de un único proveedor, cada uno utilizando diferentes protocolos. Ya sea para una configuración de hogar inteligente o fabricación automatizada, la comunicación efectiva entre sus máquinas es crucial. ROS (y, por supuesto, ROS 2) ofrece varios modos para abordar esto. ¿Qué tal si pudieras personalizar tareas mientras ROS gestiona la comunicación entre nodos?

El equipo de Robonomics ha creado varios paquetes para ROS para hacer que los robots sean más inteligentes y manejen tareas específicas según sea necesario. Para ver las tareas que hemos logrado con ROS, visita nuestra página de casos de I+D. Casi todas las interacciones con un robot dependen de ROS. La contribución de Robonomics en la comunidad de ROS 2

Robonomics combina la robótica con las tecnologías Web3, y nuestro objetivo es crecer en estas áreas. Desde la introducción de ROS 2, hemos identificado tres desafíos principales que implican apoyar a la comunidad de ROS 2 y mejorar las capacidades de Robonomics:

  • Creando herramientas ROS 2 para Rust: Estas herramientas permiten a los desarrolladores escribir aplicaciones ROS 2 usando Rust. Para descubrir más sobre por qué elegimos Rust y la importancia de este proyecto para nosotros, puedes encontrar detalles adicionales
  • Desarrollando Robonomics Middleware para ROS 2: Esto juega un papel importante en garantizar la integración perfecta de Robonomics en varios sistemas de robótica multiagente utilizando ROS 2. Puede facilitar la integración de robots en tecnologías blockchain Web 3 mientras también reduce errores durante el proceso de integración, ahorrando así tiempo y costos.
  • Creando un laboratorio de robótica con ROS 2: A medida que ROS 2 gana popularidad en la comunidad de robótica, estamos ansiosos por ver más robots y hardware trabajando con ROS 2. Nuestro objetivo es conectarlos al ecosistema Web3 utilizando Robonomics.
