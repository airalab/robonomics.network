Robonomics Photo-Booth en Decentralized Lugano 2025

smart homeevents

Muestra una canalización de medios totalmente descentralizada usando Robonomics IPFS y Home Assistant captura fotos divertidas aplica filtros de visión por computadora en tiempo real almacénalas en IPFS registra todos los hashes de medios en el Datalog de Robonomics y permite que cualquier DApp lea y muestre la galería de fotos

La Idea

1. Captura de Foto y Detección de Rostros

Se integró una cámara en el entorno de Home Assistant para capturar imágenes bajo demanda que un servicio Python embebido procesa con el clasificador Haar-cascade de OpenCV (no se requiere GPU) para detectar cada rostro cada rostro detectado se anonimiza superponiendo un emoji sonriente mediante la biblioteca Pillow

2. Integración con Home Assistant

Para agilizar el despliegue dentro de Home Assistant Alena nuestra principal colaboradora en la integración de Robonomics con Home Assistant creó un módulo ligero de integración de medios basado en la integración principal de Robonomics HA (https://github.com/airalab/homeassistant-robonomics-integration) El paquete (https://github.com/LoSk-p/robonomics_media_integration) reutiliza el conector principal para suscribirse a eventos de la cámara y publicar CIDs de meta-archivos en el Robonomics Datalog pero elimina funciones no utilizadas así que la instalación y configuración tomó solo minutos

3. Almacenamiento IPFS

Después de procesar cada imagen se fija en IPFS usando el servicio de fijado de Pinata que devuelve un identificador de contenido único CID que sirve como referencia permanente para la instantánea

4. Archivo de metadatos y datalog

Por cada nueva foto el sistema reconstruye la lista JSON de todos los CID de imágenes y fija esa lista en IPFS Una tarea en segundo plano se ejecuta una vez por minuto para comparar el CID de la última lista fijada con el registrado en el Datalog de Robonomics y si no coinciden el CID de la nueva lista se envía al Datalog para mantener la referencia en cadena actualizada

RoSeMAN Indexing

RoSeMAN nuestro indexador off-chain (también utilizado en el caso de Sensor-Map) es un servicio Node.js (ver (https://github.com/airalab/RoSeMAN) que se suscribe al Robonomics Datalog a través de un agente sensor basado en Substrate

_

En cada nuevo meta-file CID registrado en el Datalog realiza una obtención HTTP desde IPFS para recuperar y analizar la lista JSON de CIDs de fotos

_

Los CIDs analizados se insertan en el backend de MongoDB de RoSeMAN y se exponen a través de una API HTTP simple así que cualquier DApp externa puede consultar a RoSeMAN para obtener una galería actualizada de todas las fotos fijadas en IPFS

Como resultado

Los visitantes ven una galería en vivo resistente a la censura de cada sonrisa capturada en Lugano sin servidor central sin un solo punto de falla
