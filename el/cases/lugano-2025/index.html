Η Ιδέα

1. Λήψη Φωτογραφίας & Ανίχνευση Προσώπου

Μια κάμερα ενσωματώθηκε στο περιβάλλον του Home Assistant για τη λήψη εικόνων κατ απαίτηση τις οποίες επεξεργάζεται μια ενσωματωμένη υπηρεσία Python με τον ταξινομητή Haar-cascade του OpenCV (δεν απαιτείται GPU) για τον εντοπισμό κάθε προσώπου κάθε εντοπισμένο πρόσωπο στη συνέχεια ανωνυμοποιείται με την επικάλυψη ενός χαμογελαστού emoji μέσω της βιβλιοθήκης Pillow

2 Ενσωμάτωση με το Home Assistant

Για να απλοποιήσει την ανάπτυξη στο Home Assistant η Alena η κύρια συνεισφέρτρια μας για την ενσωμάτωση του Robonomics με το Home Assistant δημιούργησε μια ελαφριά μονάδα media integration βασισμένη στην κύρια ενσωμάτωση Robonomics HA (https://github.com/airalab/homeassistant-robonomics-integration) Το πακέτο (https://github.com/LoSk-p/robonomics_media_integration) επαναχρησιμοποιεί τον βασικό συνδετήρα για την εγγραφή σε συμβάντα κάμερας και τη δημοσίευση meta-file CIDs στο Robonomics Datalog αλλά αφαιρεί αχρησιμοποίητες λειτουργίες—οπότε η εγκατάσταση και η ρύθμιση πήραν μόνο λίγα λεπτά

3. IPFS Storage

Μετά την επεξεργασία κάθε εικόνα καρφιτσώνεται στο IPFS χρησιμοποιώντας την υπηρεσία pinning της Pinata η οποία επιστρέφει ένα μοναδικό αναγνωριστικό περιεχομένου CID που λειτουργεί ως η μόνιμη αναφορά για το στιγμιότυπο

4. Αρχείο Μεταδεδομένων & Datalog

Για κάθε νέα φωτογραφία το σύστημα αναδημιουργεί τη λίστα JSON με όλα τα CID εικόνων και καρφιτσώνει αυτή τη λίστα στο IPFS Μια διεργασία παρασκηνίου εκτελείται μία φορά το λεπτό για να συγκρίνει το CID της πιο πρόσφατα καρφιτσωμένης λίστας με αυτό που έχει καταγραφεί στο Robonomics Datalog και αν δεν ταιριάζουν το CID της νέας λίστας αποστέλλεται στο Datalog για να διατηρείται η αναφορά στην αλυσίδα ενημερωμένη

RoSeMAN Ευρετηρίαση

RoSeMAN η εξωαλυσίδική μας υπηρεσία ευρετηρίασης (που χρησιμοποιείται επίσης στην περίπτωση του Sensor-Map) είναι μια υπηρεσία Node.js (δείτε (https://github.com/airalab/RoSeMAN) που εγγράφεται στο Robonomics Datalog μέσω ενός πράκτορα αισθητήρα βασισμένου σε Substrate

_

Σε κάθε νέο meta-file CID που καταγράφεται στο Datalog εκτελεί ένα HTTP fetch από το IPFS για να ανακτήσει και να αναλύσει τη λίστα JSON των photo CIDs

_

Οι αναλυμένοι CIDs εισάγονται στο backend MongoDB του RoSeMAN και εκτίθενται μέσω ενός απλού HTTP API ώστε οποιαδήποτε εξωτερική DApp να μπορεί να κάνει ερώτημα στο RoSeMAN για μια ενημερωμένη συλλογή όλων των φωτογραφιών που έχουν καρφιτσωθεί στο IPFS

Ως αποτέλεσμα
