## Введение

Эта статья [расширяет интеграцию](/blog/article-ocean-p1/) между сетью Robonomics и протоколом Ocean. Здесь будет описано, как именно подключить робототехнические устройства к протоколу, а также возможные сценарии использования, где такая интеграция добавит ценности к работе автономной системы.

Для тех, кто не знаком с протоколом Ocean: идея заключается в том, чтобы превратить любые данные в полноценный актив путем токенизации, с возможностью покупки / продажи и передачи собственности. Коротко говоря, при токенизации данных вы развертываете и выпускаете токен ERC20 (дататокены), собственность на который дает вам доступ к данным, присвоенным ему (1 дататокен = доступ к набору данных). Протокол позволяет гибко управлять доступом, например, организовывать временный доступ к данным или предоставлять доступ к их аналитике без нарушения конфиденциальности. Таким образом, становится возможным монетизировать токенизированные данные, создавать для них рынки и использовать все возможности инструментов децентрализованных финансов.

Команда Robonomics отлично понимает ценность монетизации через токенизацию, но еще лучше мы понимаем ценность автоматизации. Мы уже научились автоматически монетизировать полезные действия роботов и устройств Интернета вещей, так почему бы не автоматизировать монетизацию результатов (журналов и данных) этой деятельности? С помощью протокола Ocean почти любое робототехническое устройство, генерирующее ценные данные, может быть превращено в простой источник дохода для его владельца. Или устройство может стать потребителем полезных данных для завершения своей задачи.

## Аспекты интеграции для устройств на ROS

Для того чтобы Robonomics работала с различными типами устройств, мы используем Robot Operating System, программную среду, которая позволяет удобно развертывать программное обеспечение для робототехнических устройств. Благодаря ROS, довольно легко подключить новый тип устройства к сети. Активное сообщество ROS разработало сотни готовых к использованию пакетов для различныхроботов и устройств IoT на протяжении многих лет.

Три основные абстракции в среде ROS - это узлы (nodes), темы (topics) и сообщения (messages). Узел - это процесс/программа, выполняющая определенную задачу и общающаяся с другими узлами через темы. Узел подписывается на тему, ожидая сообщения, или публикует свои собственные сообщения в них. В этой реализации компоненты робототехнической системы различных типов могут взаимодействовать друг с другом на одном уровне.

На рисунке ниже изображен типичный процесс принятия и выполнения задачи робототехническим устройством (промисором) в Robonomics v1, работающем в сети Ethereum. Блок клиента AIRA - это развернутое изображение необходимого программного обеспечения для устройства, включая ROS. На рисунке показано (в синих рамках), что устройство общается с децентрализованной сетью через клиент AIRA, получая и отправляя сообщения в соответствующих темах ROS. Миссия устройства завершается окончательной отправкой результирующего сообщения в сеть IPFS, которое содержит ссылку на данные, потенциально пригодные для монетизации.

!["Процесс Robonomics v2"](/blog/images/ocean-protocol-p-2/uml-robonomics-ver1.png)

В Robonomics v2, который разрабатывается как [парачейн](https://parachain.robonomics.network/) экосистемы Polkadot, процесс немного отличается. В настоящее время для управления устройством доступны функции `launch()` и `datalog()`, первая функция позволяет выполнять транзакции для включения/выключения устройства, а вторая отправляет datastring в блокчейн. Однако журналы и данные устройства по-прежнему собираются с использованием тем ROS. Затем их можно передать либо непосредственно в `datalog()`, либо сначала отправить в IPFS.

Таким образом, первая задача при интеграции Ocean Protocol с сетью Robonomics - это привнести API протокола в среду ROS. Далее, такой пакет для ROS можно интегрировать в клиент AIRA и автоматически развернуть создание дататокенов.

## Пример интеграции Ocean с ROS

В этом разделе предоставляется описание репозитория ocean_ros, созданного инженерами Robonomics, который является пакетом ROS, реализующим интеграцию API Ocean Protocol и ROS. Ocean Protocol предлагает несколько вариантов API для работы с ним, в данном случае используется [ocean.py](https://github.com/oceanprotocol/ocean.py) библиотека Python.используется, поскольку в ROS есть библиотека клиента Python [rospy](http://wiki.ros.org/rospy). Основная цель пакета - создать три узла ROS, через которые роботизированное устройство может развертывать и создавать datatokens, создавать пул для размещения на рынке или покупать datatokens для использования. Давайте пройдем установку и запуск пакета.

### Требования и установка

Для работы пакета требуется:

* Ubuntu
* Python 3.8.5 или более поздняя версия
* ROS, версия Melodic (инструкции по установке [здесь](http://wiki.ros.org/melodic/Installation/Ubuntu))

Установка пакета:

<RbCode>

```sh
git clone https://github.com/LoSk-p/ocean_ros
cd ..
catkin_make
source devel/setup.bash
```

### Настройка сети Ethereum и узла

В примере используется провайдер Infura для настройки соединения. Infura позволяет удаленно использовать инфраструктуру Ethereum без необходимости запуска полного узла, что очень удобно для компактного роботизированного устройства с ограниченной вычислительной мощностью.

Обратите внимание, что для примера настраивается подключение к Rinkeby, тестовой сети Ethereum. Если вам нужно переключиться на основную сеть или другую, обратитесь к [документации Ocean](https://docs.oceanprotocol.com/concepts/networks/).

Шаги для настройки следующие:

* Перейдите на https://infura.io и зарегистрируйтесь.
* На сайте Infura создайте новый проект.
* На странице настроек проекта запишите значение вашего идентификатора проекта Infura. Оно будет использовано на следующем шаге.
* Создайте файл `config.ini` в каталоге config (пример конфигурации можно найти в файле `config_template.ini`).
* Запишите ваш идентификатор проекта Infura для сети в файле конфигурации.

### Создание дата-токена

Для создания дата-токена вам необходимо опубликовать сообщение в тему `/ocean/create_datatoken`. Сообщение должно содержать путь к файлу приватного ключа из вашего аккаунта Metamask, IPFS URL для данных с устройства, OCEAN и ликвидность дата-токена. Пример сообщения:

<RbCode>

```sh
rostopic pub /ocean/create_datatoken ocean_ros/Metadata "private_key_path: '/home/user/private_ocean'                     
data_created: '2021-12-28T10:55:11Z'
type: 'dataset' 
name: 'test_ros'
author: 'author'             
license: 'CC0: Public Domain'    
files_content_type: ['text/text']
files_url: ['https://gateway.ipfs.io/ipfs/QmWRjFcQi4Xcisqi8FP3AbGS3PB3gNHgtnfzbcpodKKCBP']                 
tokens_nomber: 100.0
ocean_amount: 10.0
```
</RbCode>

Соответствующий узел ROS создаст дататокен, который станет доступным для продажи. Вы можете увидеть информацию о дататокене в теме `/ocean/token_response`:

<RbCode>

```sh
rostopic echo /ocean/token_response
```

</RbCode>

</TextCard>

<TextCard orientation="vertical" alignContent="left">

### Покупка дататокена

Для покупки дататокена вам необходимо опубликовать сообщение в теме `/ocean/get_datatoken`. Сообщение должно содержать путь к файлу с закрытым ключом из вашей учетной записи Metamask и информацию о дататокене, который вы хотите купить (вы можете найти его на [рынке](https://market.oceanprotocol.com/) или в теме `/ocean/token_response`). Пример сообщения (не забудьте изменить `destination` в сообщении):


<RbCode>

```sh
rostopic pub /ocean/get_datatoken ocean_ros/BuyDatatoken "{private_key_path: '/home/user/private_ocean', destination: '/home/user/', token_address: '0x9fb21F68257F1d718d764B68b1430B6460796e42', did: 'did:op:9fb21F68257F1d718d764B68b1430B6460796e42', pool_address: '0xcF295B85ef5ADd0E513B789477C6d14eA6Bc718a'}"
```

</RbCode>

Вы можете увидеть путь к загруженному файлу данных в теме `/ocean/buying_response`:

<RbCode>

```sh
rostopic echo /ocean/buying_response
```

</RbCode>

## Сценарии использования

Этот раздел описывает потенциальные примеры применения автоматической токенизации данных, полученных робототехническими устройствами.

### Данные об окружающей среде

Возможно, наиболее интересным примером применения была бы токенизация данных об окружающей среде. Стоимость таких данных только растет, принимаяучитывая увеличивающийся интерес населения, компаний и регуляторов к проблемам экологического управления. Основные проблемы здесь - это масштаб экологии (воздух, вода, почва, умноженные на огромные и часто недоступные территории Земли) и вопрос доверия к собранным данным. Мобильное автоматическое устройство (воздушный или водный дрон) или сеть стационарных датчиков идеально решают первую проблему, а блокчейн и децентрализованные технологии смягчают вторую. С возможностью токенизации этих данных с использованием Ocean Protocol открываются совершенно новые рынки, где идеально вписываются концепции гражданской науки и углеродных кредитов.

Robonomics долгое время работает с устройствами для экологии. Например, был разработан пакет [sensors-connectivity](https://github.com/airalab/sensors-connectivity), который позволяет превратить сенсоры, совместимые с ROS, в телеметрические агенты. Инженеры Robonomics разрабатывают различные модели [водных дронов](https://blog.aira.life/water-drone-2-0-first-stage-of-development-db8ad6c81d3c) для сбора химических показателей в реках и озерах, а также изучают возможность использования [БПЛА](http://www.ijeetc.com/uploadfile/2019/1009/20191009035041545.pdf) для оценки качества воздуха.

<RbCode>

```
1556199372.384576 <=>\x86##5E1567057C105409#DOV01SW#0#WT:6.53#PH:8.89#DO:90.4#COND:348.9#
```

</RbCode>

Пример собранных данных о воде: температура, значение PH, растворенный кислород, проводимость
В каждом случае устройства генерируют журналы сенсоров и GPS, которые затем [публикуются](https://gateway.ipfs.io/ipfs/QmWRjFcQi4Xcisqi8FP3AbGS3PB3gNHgtnfzbcpodKKCBP) в IPFS и предоставляются устройством в результате. Здесь мы можем добавить ценность этим данным, создавая дататокен с помощью Ocean Protocol. Сеть Robonomics гарантирует, что данные были собраны конкретным роботом для определенной задачи, а Ocean Protocol обеспечит доступ к ним только тем, кто владеет дататокеном.

### Маркетинговые данные

Тенденция робототехники в повседневной жизни растет, и довольно легко найти примеры роботов-пропагандистов или роботов для клиентовслужба. Опыт взаимодействия таких роботов с людьми можно использовать для маркетинговых исследований, аналогично сайтам, собирающим данные о пользователях. Более того, ценность этих данных будет еще выше, потому что взаимодействие с человеком происходит на более близком уровне.

Можно представить робототехническое устройство, которое в процессе выполнения своей конкретной задачи (например, робот-официант) одновременно собирает визуальную, звуковую или другую информацию о клиенте. В этом случае для создания совершенно нового типа данных о реакции клиента на определенный вид услуги или продукта можно использовать инструменты машинного обучения. Это отличная возможность использования дататокенов как для необработанных данных от роботов, так и для тех, которые проанализированы алгоритмами. Более того, у протокола Ocean есть функция Compute-to-data, которая позволяет предоставлять конфиденциальные данные для аналитики без нарушения конфиденциальности.

### Технические журналы

Улучшение алгоритмов роботов также требует анализа их рабочих данных. Это касается в первую очередь машинного обучения, которое требует наборов данных, но можно также говорить о совместном использовании трехмерных карт (например, для одновременной локализации и построения карты) между устройствами, работающими в одной и той же или похожей среде. Монетизация таких технических журналов будет более специфичной для отрасли, но ценность будет заключаться в их уникальности и редкости.

Примеры включают токенизацию данных, созданных роботами-пылесосами для улучшения их производительности (обход препятствий, лучшее распознавание окружения в квартирах). Другим примером являются данные обслуживания линии производства, обработка которых оптимизирует операционные или производственные издержки.

## Заключение

Приведенные примеры показывают, что сочетание сети Robonomics и протокола Ocean создает фундаментально новый тип активов, ранее недоступных из-за недостаточной автоматизации технических процессов и невозможности безопасного контроля и обмена их результатами. Более того, это только очевидные идеи, и в будущем, с распространением интернета вещей и роботов, могут появиться совершенно новые рынки для кибер-физических данных.
