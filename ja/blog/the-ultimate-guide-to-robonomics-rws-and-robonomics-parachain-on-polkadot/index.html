Polkadot上のRobonomics、RWS、Robonomicsパラチェーンの究極ガイド

February 25, 2021 in Robonomics parachain , RWS token , Subscription / All posts

本記事では、Robonomicsプラットフォーム、そのPolkadotエコシステム上のパラチェーン、そしてRobonomics Web Services（RWS）との関係性について掘り下げます。

Robonomicsの基盤運用に関する絶え間ない議論を踏まえ、内部構造を調査し、ネットワークの主要な柱を概説しましょう。本記事では、Robonomicsプラットフォーム、そのPolkadotエコシステム上のパラチェーン、およびRobonomics Web Services（RWS）との関係性について掘り下げます。

Robonomics Networkは、完全なIoTアプリケーションの実装に個別に利用できる複数のコードリポジトリで構成されています。たとえば、スマートフォンをIoTデバイスに接続しようとする場合、アプリを開発してセンサーをいくつか接続するだけでは不十分です。その代わりに、最終的にIoTデバイスが接続され、インフラの要素同士が相互接続されるような一連のアクションを実行する必要があり、これによってデバイスからの信号があなたに届くようになります。

Robonomics Network開発の年表

では、RobonomicsパラチェーンおよびRobonomics Networkを作るというアイデアはどのようにして生まれたのでしょうか？もともと、2015年から2017年にかけて、私たちはかなり複雑な開発から始めました。私たちが「ロボット経済」と呼ぶソリューションを開発しました。これこそが、スマートシティ、インダストリー4.0、そしてIoTアプリの発展が行き着く先であると確信しています。

私たちの主な目標は、技術的および経済的な取引の枠組みの中で、自律的かつ独立したデバイスを管理できるサイバーフィジカルシステム向けのソフトウェアを作成することでした。そのため、技術と経済に偏ったトランザクションを生成し、それをより複雑なプロセス開発の基盤とすることに決めました。

私のお気に入りのロボット経済のシナリオの一つは、スマートビルディング・オフセッティングと呼ばれるものです。これは、自律的な建物が自らのエネルギー消費に関する情報を収集・集約し、それをロボノミクスネットワーク内の別のエージェントに伝達し、さらにはカーボンフットプリントのオフセットサービスを依頼できるサービスです。一方、エージェントは建物の所在地を確認し、CO2排出量の適用係数を算出し、このデータをパートナープロジェクトDAO IPCIと同期させ、建物のカーボンフットプリントを相殺するために必要なグリーンファイナンス市場からのカーボンクレジット数を決定します。これらの操作はイーサリアムブロックチェーン上で正常に実行されました。

問題の解決策

今日、私たちが2017年に同僚やパートナーと共に構築したロボット経済は、一見複雑なチェーンで構成されており、IoT業界進化の最終段階として機能しています。最大の課題は、私たちが知識を蓄積し、アイデアを実現してきたにもかかわらず、それを他の誰かに伝える機会がなかったことです。ロボット経済の新しいシナリオがロボノミクスネットワークプラットフォームの開発中に生まれるたびに、最終製品はこの分野に不慣れな人々にとって理解しづらいものになっていきました。言い換えれば、複数の専門分野に精通し、経済と機械が協働するという概念を形成する学際的な思考を理解するには、何年もかけて習得する必要があるのです。

IoTアプリケーション開発の裏にあるニュアンスを理解し始めると、技術者が完全に機能する製品段階に到達するまでに何年もかかることがすぐに明らかになります。だからこそ、私たちはプラットフォームの潜在的なユーザーを共同開発者に変えることに決めました。彼らがIoTアプリケーション分野のプロフェッショナルなITエンジニアになるよう教育することに注力し、その結果、私たちはRobonomicsパラチェーンとRobonomics Web Servicesを作成しました。

Robonomics Web Servicesの紹介

ロボット経済が一般にとって興味深いものであることは明らかです。しかしながら、主な問題は、しばらくするとこれらのユーザーがIoTデバイスのためにAmazon Web Servicesに流れ、そちらを使い始めてしまうことです。その際、Robonomics Networkが有名なIT大手と比べて持つ重要な利点を理解しないままです。このことから、私たちは「将来何かを完成させる」という約束だけではなく、今ここで実際の製品を提供しなければならないという結論に至りました。

そのため、私たちは潜在的なユーザーに対して、実際のサービスを提供する必要があると決断しました。他のクラウドプラットフォームが提供しているものと同様、しかしそれ以上のサービスです。その結果、私たちは自社プロトコルとプラットフォーム上に構築されたRobonomics Web Servicesを開発しました。これはオープンソースのロボティクスフレームワーク（ROS）を利用しています。その後、私はこれから詳しく説明する主要な機能の開発に2年間を捧げました。

RWSの主な機能：

1. サブスクリプション

ブロックチェーンサービスの標準的なアプローチには、トランザクション手数料やガス代として知られる取引コストが常に含まれます。つまり、トランザクションを行う際、その優先度は支払う用意のあるコストや手数料に依存します。高い手数料を支払えば支払うほど、トランザクションの処理は速くなります。このようなモデルは、保証された帯域幅に依存するシステムには適していません。デバイスが設定された時間に予測可能なコストで確実にトランザクションを行えることを絶対に保証しなければなりません。今日ではこのようなことはあまり見られません。なぜなら、トランザクションコストは為替レートの変動に完全に依存しており、時には50ドルに達することもあるからです。

この問題を解決するために、私たちはサブスクリプションサービスを作成しました。これはPolkadotのRobonomicsパラチェーン上での代替的なコストモデルです。あらかじめ支払った一定数のトランザクションの実行を保証し、手数料モデルで支払われるトランザクションよりも高い優先度を割り当てます。月額サブスクリプションとXRTトークンの違いについて詳しく知りたい方は、私の『Heterogeneous tokenomics』の記事をご参照ください。私の他の記事も計画中です。12月末にリリース予定のものでは、2つのモデルの最終的なアーキテクチャと相互接続について概説します。

私たちのサブスクリプションモデルは、保証された移行問題を解決するだけでなく、1台あたりわずか10ドルという非常に収益性の高いオファーをエンドユーザーに提供します。さらに、毎月数千件のトランザクションに対して安定した帯域幅を提供しており、これは非常に優れた成果です。

2. Launch

Launchは、計算能力やブロックチェーンの観点から見ても非常にシンプルで経済的な機能です。これにより、トランザクションの形でリクエストを送信することで、デバイスの電源をオンまたはオフにしたり、バイナリ設定を1から0、またはその逆に変更したりできます。この機能には複雑さがほとんどないため、次の重要な柱に進みましょう。

3. Datalog

この機能はテレメトリーと連携して動作するため、少し詳しく説明します。2つのモードがあり、IPFSハッシュを保存するか、データをチェーン上に保存するかを選択できます。

では、これらのモードはどのように使い分けるのでしょうか？オンチェーンのデータログ機能は、データサイズが4変数未満の場合に適用されます。これほど小さなデータをIPFS（オフチェーン）に保存する意味はほとんどありません。バイト数的にも、このオンチェーンログはIPFSハッシュよりも大きくはなりません。私たちは、すべての生データの流入を1つの関数として記録するか、IPFSハッシュを選択することを推奨します。その後、Robonomicsは開発者向けプラットフォームとして、記録されたデータが生データであれIPFSハッシュであれ、それをどこに書き込むかを判断します。このデータログ処理はエンドユーザーからは隠されており、2つの異なるストレージタイプの違いに気付くことはありません。

「Liability」は私たちの最終的な機能ですが、大規模なロボット経済プロジェクトが追求する目標が明確になるまで、一般公開はしません。私たちが取り組んでいるもう1つのマイルストーン製品は、Robot as a Serviceフレームワーク、または私たちが呼ぶ「契約義務を持つロボット」をEthereum上で実現することを目指しています。このインフラストラクチャは、すでに私たちのナレッジハブで詳細に説明されており、エンドユーザーが私たちのサービス・トークンを使用する状況を想定しています。XRTや他のトークンに依存せずにパラチェーンの利用料を支払うことができます。

RWSとロボット経済の関係は？

前述の通り、Robonomics Web Servicesの目的の一つは、ユーザーを教育し、ロボット経済を理解する共同開発者へと育てることです。このタスクは、ユーザーが上記で説明した基本的な機能に慣れ親しめば、決して複雑なものではないと私たちは考えています。

デジタルツイン

Robonomics Web Servicesが提供するツールを活用する次のステップは、デジタルツイン、またはアプリのデジタルシャドウの作成方法を学ぶことです。これは現実世界の機器のデジタル版として機能し、そのすべての技術的特性をコピーします。

どういう意味かと疑問に思うかもしれません。例として、空気質を測定する複雑なセンサー構造を考えてみましょう。これらのセンサーは、内部のキャリブレーションや測定データのバランス調整に2分かかるため、5秒ごとに測定を行うことはできません。ここでデジタルツインが役立ち、ユーザーのコマンドに応答するアプリを作成できるようになります。

ガートナーのIoT市場動向レポートによると、調査対象となった企業の85%が、すでにデジタルツインを導入しているか、今後1年以内に導入を計画しています。これらの企業の多くは、社内のエンジニアリングチームが独自のソリューションを開発していますが、迅速なスケーリングや、絶えず変化するIoT業界環境への適応性を維持するためには、既存のソリューションに頼らざるを得ないことを認識しています。

このため、Robonomics Networkのパラチェーンは、Ethereum IPFSでは利用できない上記の機能だけでなく、ユーザーが自分のデバイスのデジタルツインを作成できる機能も提供しています。

私たちはこの機能を開発し、ユーザーが自分自身のデジタルツインを構築し、それをWebサービスやIoTアプリで解釈し、さらに重要なことに、データログを書き込むモデルとして保存できるようにしています。私たちのWikiは、これらのソリューションを実行するために必要な知識を随時更新しており、まもなくデジタルツインの構築方法をAからZまで説明する基本情報が揃う予定です。

これが、Robonomics Networkのパラチェーンが、Ethereum IPFSでは利用できない前述の機能だけでなく、ユーザーがデバイスのデジタルツインを作成できるようにする理由です。

ツイン

データ処理モデルが確定し、デジタルツインが完全に構築されたら、そのデジタルツインがウェブサービスやIoTアプリケーションとどのような関係を持つかを決める必要があります。

このようなデジタルサービスでは、アナリティクスが一般的な選択肢となります。センサーが測定したデータを送信し（デジタルコピーを更新する）、このデータはほとんどの場合、後処理の対象となります。なぜなら、あなたが自宅の暖房システム内のパスカル圧力値に興味があることはほとんどないからです。代わりに、家の中の温度が適切かどうか、暖房システムがまだ稼働しているかどうかを知りたいはずです。つまり、保存したデータのデジタルツインをどのように活用するかを決める必要があり、ここで追加のサービスが役立ちます。

次に、エネルギー消費データのコピーがデジタルツインとして機能する、スマートビルディング・オフセッティングの複雑なシナリオに移ります。たとえば、特定の部屋のエネルギー消費データを保存し、そのためにIPFSストレージを使ったDatalog機能を利用したいとします。消費されたエネルギーの合計量を知りたい場合は、それをオンチェーンに保存します。逆に、スマートビルディングのデジタルツインを運用していると想像してください。オフセッティングサービスは、このデジタルツインに頻繁にエネルギー消費に関するさまざまなデータを提供するよう要求し、それを保存し、オフチェーン計算を行った後、必要なサービスを提供するためにRobonomicsと接続・通信します。

このようなサービスは、ロボットのためのApp Storeとして機能する専門のマーケットプレイスに集約されています。

マーケットプレイスの理解

相互接続されたデジタルツインとサービスのモデルを実装するには、それを届けるための媒体、つまりスマートフォンのアプリを作成する必要があります。このマーケットプレイスは、分散型アプリケーションという形で分散型台帳上に構築されており、あらかじめ定義されたテンプレートを使って、あらゆる目的のために個別のコピーやデジタルツインを複製・作成することができます。最も重要なのは、仲介者や外部ストレージを介さずにこれが実現できるため、IoTサービスのマーケットプレイスが検閲や中央集権的なリンクから解放されていることです。

最後に、ユーザーインターフェースについてお話ししましょう。私たちは常に最新のオンライン技術をユーザーに提供することを目指しており、そのため分散型アプリケーションへのアクセスを提供しています。開発中に得られる最大の特徴はあなたのDappにおけるもう一つの利点は、あなた側に脆弱性がないことです。あなたのアプリがIPFS上にあっても、ENSレジストリやPolkadotエコシステムレジストリに接続されているため、ユーザーはハイパーリンクをたどる際に置き換えられたアプリやリクエストに対処する脆弱性に直面しません。これはエンドユーザーアプリケーションの開発や保守の観点から非常に便利です。

分散型アプリケーションのもう一つの利点は、ピアツーピア通信です。つまり、あなたのアプリはデジタルツイン、IoTサービス、エンドユーザーアプリケーションと単一障害点なしに通信でき、タスクを実行する際に第三者や仲介リソースを必要としません。これがrobonomics.jsの役割であり、Dapp Robonomicsの開発に使われています。現在、私たちは新しい改良バージョンの追加作業中で、まもなくRobonomics Stylebook（Web3.jsとpolkadot.jsの機能に基づくインターフェースガイド）をリリース予定です。これにより、あなたはアプリケーションの構造化を簡単に行うことができます。その結果、ロボット経済の完全なチェーンが完成します。

まとめ

  • 私たちの目標はロボットを経済の一部にすることですが、まずはユーザーに仕組みやセットアップ方法を教える必要があります。
  • その目的のために、私たちはRobonomics Web Servicesという製品を作成しました。これにより、ユーザーは私たちと一緒にアプリやサービス（サブスクリプション、起動、データログ）を構築できます。
  • RWSへのサブスクリプションは、すべてのトランザクション関連の問題を解決し、デバイスのオン/オフ、テレメトリ（デジタルツインの保存）などの機能を追加します。
  • 社内テンプレートを使ってデジタルツインを作成したら、IoTサービスを開発し、私たちのマーケットプレイスに追加できます。Dapp内の事前構造化されたインターフェースは、あなたのサービスをエンドユーザーに届ける仕組みとして使われます。
  • Dapp は、サービスが停止した場合でも、デジタルツインとのピアツーピア（直接）通信を確立するための堅牢な方法として機能します。外部の状況に関係なく、市場は決して機能を失うことはありません。
  • 前述のチェーンは、本記事で説明されているロボット経済への道筋です。
