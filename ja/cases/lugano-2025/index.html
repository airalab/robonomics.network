ロボノミクスフォトブースアットディセントラライズドラガーノ2025

smart homeevents

Robonomics IPFS Home Assistantを使って完全に分散化されたメディアパイプラインを披露する funnyな写真を撮影しオンザフライでコンピュータビジョンフィルターを適用しIPFSに保存しすべてのメディアハッシュをRobonomics Datalogに登録しどのDAppでもフォトギャラリーを読み取り表示できるようにする

アイデア

1. 写真撮影と顔検出

カメラがHome Assistant環境に統合されオンデマンドで画像をキャプチャし組み込みのPythonサービスがOpenCVのHaarカスケード分類器（GPU不要）で全ての顔を検出します検出された各顔はPillowライブラリを使って笑顔の絵文字を重ねることで匿名化されます

2. Home Assistantとの統合

Home Assistant内でのデプロイを効率化するためにRobonomicsとHome Assistantの統合の主な貢献者であるAlenaがメインのRobonomics HA統合に基づいた軽量な“media integration”モジュールを作成しました (https://github.com/airalab/homeassistant-robonomics-integration) The package パッケージ (https://github.com/LoSk-p/robonomics_media_integration) カメライベントの購読とメタファイルCIDのRobonomics Datalogへの公開のためにコアコネクタを再利用しますが未使用の機能は削除されているためインストールと設定は数分で完了しました

3. IPFS Storage

処理後 each image is pinned to IPFS using Pinata’s pinning service which returns a unique content identifier (CID) that serves as the permanent reference for the snapshot

4. メタデータファイル & データログ

新しい写真ごとにシステムはすべての画像CIDのJSONリストを再構築しそのリストをIPFSにピンします バックグラウンドタスクが1分ごとに実行され最新のピン済みリストのCIDとRobonomics Datalogに記録されているものを比較します もし一致しなければ新しいリストのCIDがDatalogに送信されオンチェーンリファレンスが最新の状態に保たれます

RoSeMAN インデキシング

RoSeMAN our off-chain indexer (also used in the Sensor-Map case) is a Node.js service (see (https://github.com/airalab/RoSeMAN) Substrateベースのセンサーエージェントを介してRobonomics Datalogを購読する

_

各新しいメタファイルCIDがDatalogに記録されるたびにIPFSからHTTPフェッチを実行して写真CIDのJSONリストを取得および解析します

_

解析されたCIDはRoSeMANのMongoDBバックエンドに挿入されシンプルなHTTP APIを通じて公開されるため外部のDAppはIPFSにピン留めされたすべての写真の最新ギャラリーをRoSeMANに問い合わせることができます

その結果

訪問者はルガーノで捉えられたすべての笑顔のライブで検閲に強いギャラリーを見ることができます中央サーバーも単一障害点もありません
