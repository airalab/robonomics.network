Robonomics Photo-Booth op Decentralized Lugano 2025

smart homeevents

Toon een volledig gedecentraliseerde mediapijplijn met Robonomics IPFS en Home Assistant leg grappige foto's vast pas direct computer vision filters toe sla ze op IPFS op registreer alle mediahashes op de Robonomics Datalog en laat elke DApp de fotogalerij lezen en weergeven

Het idee

1. Foto vastleggen & gezichtsdetectie

Er werd een camera geïntegreerd in de Home Assistant omgeving om op aanvraag beelden vast te leggen die door een ingebouwde Python service worden verwerkt met de Haar-cascade classifier van OpenCV (geen GPU vereist) om elk gezicht te detecteren elk gedetecteerd gezicht wordt vervolgens geanonimiseerd door een glimlachende emoji te plaatsen via de Pillow bibliotheek

2. Integratie met Home Assistant

Om de uitrol binnen Home Assistant te stroomlijnen heeft Alena onze belangrijkste bijdrager aan de Robonomics integratie met Home Assistant een lichtgewicht media integratie module gemaakt gebaseerd op de hoofd Robonomics HA integratie (https://github.com/airalab/homeassistant-robonomics-integration) Het pakket (https://github.com/LoSk-p/robonomics_media_integration) hergebruikt de core connector voor het abonneren op camera events en het publiceren van meta-file CIDs naar de Robonomics Datalog maar verwijdert ongebruikte functies dus installatie en configuratie kostte slechts enkele minuten

3. IPFS Opslag

Na verwerking wordt elke afbeelding vastgezet op IPFS met behulp van de pinningservice van Pinata die een unieke content identifier (CID) retourneert die dient als de permanente referentie voor de snapshot

4. Metadata File & Datalog

Voor elke nieuwe foto bouwt het systeem de JSON lijst van alle image CIDs opnieuw op en pint die lijst naar IPFS Een achtergrondtaak draait één keer per minuut om de CID van de laatst gepinde lijst te vergelijken met die welke is vastgelegd in de Robonomics Datalog en als ze niet overeenkomen wordt de CID van de nieuwe lijst naar de Datalog gestuurd om de on chain referentie up to date te houden

RoSeMAN Indexering

RoSeMAN onze off-chain indexer (ook gebruikt in de Sensor-Map case) is een Nodejs service (zie (https://github.com/airalab/RoSeMAN) die zich abonneert op de Robonomics Datalog via een op Substrate gebaseerde sensoragent

_

Bij elke nieuwe meta-file CID die in de Datalog wordt geregistreerd voert het een HTTP fetch uit van IPFS om de JSON lijst van foto CIDs op te halen en te parseren

_

Geparste CIDs worden ingevoegd in de MongoDB-backend van RoSeMAN en beschikbaar gesteld via een eenvoudige HTTP API zodat elke externe DApp RoSeMAN kan raadplegen voor een up-to-date galerij van alle foto's die op IPFS zijn gepind

Als gevolg

Bezoekers zien een live censuurbestendig galerij van elke glimlach vastgelegd in Lugano geen centrale server geen enkel storingspunt
