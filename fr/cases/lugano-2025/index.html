Robonomics Photo-Booth à Decentralized Lugano 2025

smart homeevents

Présentez un pipeline média entièrement décentralisé utilisant Robonomics IPFS et Home Assistant capturez des photos amusantes appliquez des filtres de vision par ordinateur à la volée stockez-les sur IPFS enregistrez tous les hachages des médias sur le Datalog de Robonomics et laissez toute DApp lire et afficher la galerie de photos

L idée

1. Capture de photo et détection de visage

Une caméra a été intégrée dans l’environnement Home Assistant pour capturer des images à la demande qu’un service Python embarqué traite avec le classificateur Haar-cascade d’OpenCV (aucun GPU requis) pour détecter chaque visage chaque visage détecté est ensuite anonymisé en superposant un emoji souriant via la bibliothèque Pillow

2. Intégration avec Home Assistant

Pour simplifier le déploiement dans Home Assistant Alena notre principale contributrice à l'intégration de Robonomics avec Home Assistant a créé un module léger d'intégration média basé sur l'intégration principale de Robonomics HA (https://github.com/airalab/homeassistant-robonomics-integration) Le paquet (https://github.com/LoSk-p/robonomics_media_integration) réutilise le connecteur principal pour s’abonner aux événements de la caméra et publier les CIDs des méta-fichiers dans le Datalog de Robonomics mais supprime les fonctionnalités inutilisées—donc l’installation et la configuration n’ont pris que quelques minutes

3. Stockage IPFS

Après traitement chaque image est épinglée sur IPFS en utilisant le service d’épinglage de Pinata qui renvoie un identifiant de contenu unique CID qui sert de référence permanente pour la capture d’écran

4. Fichier de métadonnées et Datalog

Pour chaque nouvelle photo the system rebuilds the JSON list of all image CIDs and pins that list to IPFS A background task runs once a minute to compare the CID of the latest pinned list with the one recorded in the Robonomics Datalog—and if they don’t match the new list’s CID is sent to the Datalog to keep the on-chain reference up to date

RoSeMAN Indexation

RoSeMAN notre indexeur hors chaîne (également utilisé dans le cas de Sensor-Map) est un service Node.js (voir (https://github.com/airalab/RoSeMAN) qui s’abonne au Robonomics Datalog via un agent capteur basé sur Substrate

_

À chaque nouveau CID de méta-fichier enregistré dans le Datalog il effectue une récupération HTTP depuis IPFS pour récupérer et analyser la liste JSON des CID de photos

_

Les CIDs analysés sont insérés dans le backend MongoDB de RoSeMAN et exposés via une simple API HTTP donc toute DApp externe peut interroger RoSeMAN pour une galerie à jour de toutes les photos épinglées sur IPFS

En conséquence

Les visiteurs voient une galerie en direct résistante à la censure de chaque sourire capturé à Lugano aucun serveur central aucun point de défaillance
