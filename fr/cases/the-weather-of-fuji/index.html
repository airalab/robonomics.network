Démonstration "Obtenir la météo du mont Fuji"

ethereumsmart service

Envoyez un message de demande et obtenez le résultat de l'agent autonome. Le fichier résultant est enregistré sur IPFS et le message résultant nécessite une signature avec une clé privée.

Aperçu

Cette étude de cas explore comment Robonomics Network facilite l'accès décentralisé aux données en utilisant des appareils IoT et la technologie de la blockchain. En exploitant l'application décentralisée Robonomics connectée à un capteur météorologique sur le mont Fuji, les utilisateurs peuvent obtenir des données météorologiques en temps réel de manière sécurisée et transparente.

Comment ça fonctionne?

Exigences:

  • Extension MetaMask (Téléchargez ici)

Étapes pour récupérer les données météorologiques:

  1. Ouvrez le dApp: Visitez le Capteur météo Fuji page dans l'application Robonomics.

  2. Installer MetaMask (si nécessaire): Si MetaMask n'est pas installé, vous serez invité à l'installer pour que la dApp fonctionne correctement.

  3. Autoriser la connexion: Connectez votre portefeuille MetaMask à l'application Robonomics lorsque vous y êtes invité.

  4. Demander les données du capteur: Cliquez sur le bouton "Demander les valeurs actuelles" pour initier la demande de données.

  5. Signer le message: Confirmez la demande de données en signant un message via MetaMask. Aucun jeton ou éther n'est requis.

  6. Processus de collecte de données: Un agent autonome collectera les données météorologiques du capteur du Mont Fuji et les transmettra. Attendez la réponse.

  7. Recevoir le résultat: Le dApp téléchargera le fichier de résultat depuis IPFS (InterPlanetary File System), une solution de stockage décentralisée.

  8. Voir les données météorologiques: Les données météorologiques récupérées du mont Fuji seront affichées dans l'interface de l'application décentralisée.

Aperçu technique

  • Communication décentralisée: La demande de données météorologiques est traitée en utilisant le protocole Robonomics, garantissant une communication transparente sans intermédiaires.
  • Stockage IPFS: Les résultats sont stockés de manière sécurisée sur IPFS, avec une signature cryptographique garantissant l'intégrité des données.
  • Agents autonomes: Les contrats intelligents et les agents autonomes facilitent la collecte et la livraison sécurisées de données.

Avantages clés

  • Récupération de données sans confiance: Aucune autorité centrale n'est nécessaire pour valider les données.
  • Sécurité: Les données sont cryptographiquement signées à l'aide de la clé privée de l'agent.
  • Transparence: Toutes les interactions et les résultats sont stockés de manière immuable sur la blockchain et IPFS.
