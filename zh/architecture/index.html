Robonomics架构

Robonomics的作者试图传达的主要观点是，从当前经济水平向完全自动化服务的自然、逻辑和看似不可避免的演变。自动化服务需要具备能够讲述人类经济语言和机器自身经济的能力，以便在世界各地进行各种操作，速度远远超过人类所能达到的。

接下来，我们想向您展示使用现代互联网技术的人机交互的实际领域。我们将从熟悉的物联网设备架构中的技术信息交换，通过云服务，到通过一个去中心化、全球可访问的服务提供商网络组织技术经济交互。

首先，我们需要深入探讨与自动化系统的网络连接最相关的领域。之后，我们将研究物联网的基础知识，以突出物联网系统创建者面临的问题和主要挑战，以及Robonomics所致力于解决的问题。

物联网（IoT）的现状概述

自上世纪60年代以来，全球各地一直在讨论自动设备的网络可访问性问题，当时已经有相当严肃的尝试来解决这个问题。例如，苏联的控制论专家维克托·格鲁什科夫提出并部分实施了自动收集个别工厂生产量信息的设计。 OGAS 在60年代和70年代，美国建立了国家信息系统。众所周知，可口可乐自动售货机在1980年代与ARPANET连接，以便自动售货机可以报告饮料的供应情况和温度。

然而，由于启用技术的广泛应用，物联网的实际发展正在今天发生。物联网应用快速演进的主要驱动因素是许多领域的发展成果——从各种范围的无线通信和嵌入式设备，到云分析和机器学习。过去10年来，物联网设备的数量一直在稳步增长（预计到2030年将达到250亿台），这促使研究人员和开发人员在医疗保健、家庭自动化、农业业务、环境保护等各个领域创建新的物联网应用。

我们对构建物联网系统的过程以及在此过程中出现的任务更感兴趣。多年来，已经提出了许多不同的架构，通过对它们进行平均，我们可以区分物联网系统的三个基本组成部分：设备、网络基础设施、分析、计算和控制机器。

新挑战

物联网的发展和应用表明，解决典型问题已经不够了。基于早期错误和已识别的缺陷，出现了全新的概念和原则，这些概念和原则已经在当今得到了发展。使用云服务的这种方便方法实际上存在一些与可扩展性、隐私和安全性相关的关键缺陷。为了解决这些问题，提出了将数据处理、逻辑和计算更接近于它们被使用的地方的想法。因此，雾计算和边缘计算的概念应运而生，中间系统介于云和设备之间，承担了云的一些功能。因此，创建边缘系统已经成为构建物联网的新挑战之一。

近期建立物联网的另一个挑战是数字孪生的概念——实际设备的数字版本，复制其规格和历史数据。这个概念在需要准确和完整的过程规格以及能够预测其发展的领域中表现最佳。物联网充当数字孪生的主要整合者，但这项任务比物联网设备的通常连接更为深刻。

我们目前正在见证一个整个网络部分，这个部分设计得非常方便，但不是为了人类，而是为了机器。物联网不再仅仅是将设备连接到网络的机械过程，而是更多的服务，专门为设备创建和提供。物联网不仅连接用户和设备，还帮助设备处理为人提供服务的任务，但是自主地。

物联网设备的分类

嵌入式物联网系统

这种系统没有用户界面，通常用于物联网生态系统的最低级别，提供基本功能。必须具有高能效和小型化。具有有限的计算和通信资源。物联网网关或控制器通常充当信号接收器。

边缘系统：物联网网关和控制器

这些系统充当本地可访问的嵌入式系统与全球互联网之间的链接。它们通常支持嵌入式系统的多种通信协议，并将设备的遥测传输到云端，并将控制信号传回。通常，这些设备是WiFi路由器的更复杂版本，配备了单板计算机，用于在不离开本地网络的情况下对来自设备的信息进行主要处理。

自主系统

这个类别包括复杂的自动机（比如波士顿动力的Spot），它们具有一个控制组件，相当于一个配备高性能处理器和大量内存的完整计算机。在这种情况下，机器人连接到互联网而无需额外的中间解决方案（边缘系统）。用于访问物联网的必要软件直接安装在独立系统的计算机上。数据在本地部分收集和处理，然后像任何完整的网络设备一样广播到互联网上。

物联网服务提供商

物联网服务提供商的主要工作是为用户提供远程访问数据和分析，以及通过互联网管理智能设备。服务提供商质量的主要标准是运营安全性、连接稳定性以及全球范围内的可用性。

与物联网提供商合作的标准方案包括：安装与云智能设备兼容的软件，在设备首次成功连接后设置云端管理服务，连接各种类型的分析服务。

数字孪生是物联网的核心

构建物联网解决方案的当前问题

全球可用性

标准的客户端-服务器架构要求在从物联网设备路由消息时指定端点。因此，在智能设备的使用寿命中，缺乏连接将意味着在控制和与设备交互方面存在限制。

例如，中美之间的贸易战已经是悬在美国数千名中国智能家电消费者头上的达摩克利斯之剑，反之亦然。在任何时刻，美国当局封锁小米服务器可能会影响连接到该制造商云服务的设备用户。

如果与物联网设备在附近，且没有连接到服务器，那么您无法通过移动应用程序控制它，因为用户身份验证和从设备本身收集遥测数据都是通过服务器提供的。换句话说，设备实际上是由云控制的，而不是由用户控制的。

网络安全

大多数物联网解决方案安全性低的最重要原因源自云客户端服务器架构的问题遗留。例如，目前最常见的智能家居架构通常是一个物联网中心，连接到一个云服务。因此，一个典型的物联网系统是一个连接到多个外部系统的中央控制元素。当然，云服务比低级设备更安全，但对数十万这样的系统进行集中控制使得任何安全漏洞和泄漏都成为一场巨大的灾难。

云服务服务器面临着所有典型的网络安全问题：网络攻击、DoS攻击（使系统崩溃）、恶意修改、用户会话劫持等。但强大的威胁也包括意外问题，如设计错误、误用或自然灾害导致服务器故障。另一个绝对重要的问题是基本上需要信任云和维护它的人。

所有这些都受到物联网解决方案规模的加剧影响。物联网系统的数量。当然，云服务比低级设备更安全，但对数十万这类系统进行集中控制使得任何安全漏洞和泄漏都成为一场巨大的灾难。

数据隐私

隐私问题需要与网络安全分开，因为这个问题是特定于物联网系统的。由于物联网系统中设备类型众多且多样，比起仅仅是计算机和智能手机，这些设备更加融入了物理世界，私人数据的类型也更加多样。现在不仅仅是GPS位置、购物记录、金融交易和照片，还有视频和对话的永久记录、语音印记、健康状况，甚至一个人的习惯清单。与隐私相关的威胁类型包括：信息识别、定位和追踪、个人资料编制、库存攻击、数据链接攻击。

最令人毛骨悚然的部分是用户很少或根本没有意识到他们正在放弃隐私，因为他们几乎不知道正在收集什么数据以及如何使用这些数据。隐私受到威胁的原因不仅来自入侵者，还来自组织和政府机构，正是因为这个原因。云IoT服务通常需要从设备进行最大数据传输，而且没有办法禁止或限制数据传输（“接受条款或滚蛋”）。此外，尽管收集的数据是一种资产并具有真正的价值，但被收集数据的个人不仅被剥夺了积极处理数据的权利，甚至连对自己数据的收集和使用都没有任何补偿。

Robonomics架构的主要任务

在云平台的成就和挑战的基础上，Robonomics旨在为物联网市场提供更安全、更先进的互联网解决方案，以改善人机沟通的每个阶段。以下是Robonomics致力于解决的主要任务：

1 >>

使用分散式云管理物联网设备

区块链网络具备提供设备运行和获取遥测数据的最高安全度的所有能力。与设备的交互实际上可以通过改变存储在区块链网络中的数字孪生状态并发送有关这些更改的信息来描述。在无需许可的区块链情况下，我们可以谈论物联网提供者的全球可用性保证，以及对数字孪生数据免受未经授权更改的高级保护水平。

2 >>

在人与机器之间进行技术和经济交易

由于加密货币和智能合约的机制，我们可以直接为数字市场提供对自动化系统的访问。例如，以太坊已经实现了大部分基本的经济活动工具，可以用于自主设备的运作。为了与终端用户和物联网设备之间的互动而创建的应用程序，如果服务条款和付款与设备的启动参数不可分割，那么将会更加有效。

3 >>

用户的无服务器物联网应用程序

Web3 技术的现代进步可以解决用户隐私问题。使用 web3，无需验证或连接到特定服务器即可访问遥测数据并控制设备。

人机界面:

通信协议:

它的角色由Robonomics网络扮演，这是项目团队开发的一个规范，实现了一个提供传输、处理和存储运行物联网设备所需信息的去中心化提供者网络。事实上，它是IPFS、以太坊和Polkadot网络的功能特定节点的实现。

机器界面:

我们选择了最知名的机器人操作系统（ROS）作为集成任何兼容的网络物理系统的标准接口。Robonomics支持ROS的多种用途。Robonomics在以太坊网络上的实现形式是ROS包robonomics_comm。对于ROS 2架构，我们将Robonomics支持实现为中间件层（详见《Robonomics作为ROS 2中间件》）。当使用ROS是多余的时候（例如，与物联网设备一起工作时），已经开发了一个Python robonomics-interface模块。它有助于本地和私有节点以及远程和公共节点访问Robonomics的功能。

Robonomics架构的简单表示

这是使用Robonomics平台的完整设备启动循环的一般图表：

  1. 用户的去中心化应用程序通过Robonomics网络发送设备控制功能的请求。
  2. Robonomics网络利用libp2p提供请求转换功能，并在以太坊区块链或Robonomics平行链中捕获事件。
  3. 设备的控制由一个分散式网络提供，该网络保证存储了运行物联网设备所需的必要信息。
  4. 物联网设备独立地从以太坊和Polkadot网络接收有关数字孪生当前状态的信息。这确保了设备只有在技术和经济启动参数已经确定后才能安全启动。
  5. 由于操作的结果，物联网设备会发布一个已完成操作的日志，这可以作为提供服务的证据或用于调试错误。
  6. 日志被转换为用户报告，并在他的dapp中可用。
